# v0.3 Audio Note Capture

This backend slice adds a small audio upload surface for future note-linked
audio capture.

## Backend API

`POST /api/notes/audio` accepts multipart audio uploads for:

- `audio/webm`
- `audio/wav`
- `audio/mpeg`
- `audio/mp4`
- `audio/ogg`

Uploaded files are stored under `public/uploads/audio/`. Audio metadata is
recorded separately from image assets in `runtime/asset-index/note-audio.jsonl`.

The response includes:

- `audio_id`
- `public_url`
- `mime_type`
- `size_bytes`
- `sha256`

## Non-goals

This slice does not add the frontend recording UI. It also does not change the
existing image paste upload flow, image magic-bytes validation, or image JSONL
asset index.
